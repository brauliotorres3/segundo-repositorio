¿QUE ES GIT?

Sistema de control de versiones (archivos) Controlar los cambios de un repositorio. Base de comandos que nos permiten gestionar los cambios

que es un repositorio

Es una carpeta carpeta que contiene archivos (proyecto) Biblioteca de códigos y tenerlos guardados y para guardar los cambios

Guardar los proyectos

que es GITHUB

Plataforma en internet que se basa en GIT, ver gráficamente lo que está sucediendo con muestro repositorio (commits, branchs, versiones, archivos, codigo de cada acrivo) Para compartirlos con otros usuarios

Githib nos genera un perfil para cada usuario con un m=nombre de usuario.

que es GITHUB

Para ejecutar los comandos, necesito estar ubicado en el terminal o consola

  • git

    Base para ejecutar comandos de git

  • git init

    Inicializar un repositorio de github en la carpeta o ruta actual

  • git add

    [archivos a agregar]: Agregar cambios a mi repositorio, si pongo * agrega todos los cambios actuales

  • git commit -m

    [mensaje que indica los cambios efectuados]: Confirmar los cambios en el repositorio (Esto genera a nivel interno en git una rama o Branch)

  • git push origin

    [rama origen]: Enviar los cambios confirmados a github.

  • git log

    Ver el log de todos los cambios que hemos confirmado o commits

  • git status

    ver el estado actual de los archivos en mi repositorio

  • git branch

    [nombre de la rama]: crear una rama de trabajo a partir de la rama actual. git checkout [n…

  • git checkout

    [nombre de la rama]: ubicamos en la rama para hacer cambios en ella

  • git commit --amend

    agrega cambios al ultimo commit realizado

  • git checkout -b

    Crea la rama y se ubica en ella

  • git puhs --force

    enviar cambios al servidor de manera forzada

    esto sobreescribe los cambios en la rama
